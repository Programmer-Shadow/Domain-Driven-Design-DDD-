关键领悟： Order 域的 CustomerInfo 和 User Service 的 UserResponse 是完全独立的两个模型。 即使 User Service 未来把 username 改成 displayName，Order 域的业务代码一行都不用改 — 只需修改 ACL 翻译层中的 UserTranslator。这就是防腐层的价值。

DDD 核心概念速查

限界上下文 (Bounded Context)

每个微服务是一个限界上下文，有自己的通用语言。User Service 里叫 "username"，Order Service 里叫 "customerName" — 两个上下文对同一概念有不同的命名和理解。ACL 负责在边界处翻译。

值对象快照 (Value Object Snapshot)

CustomerInfo 是下单时刻客户信息的快照，不是引用。用户之后改名，订单里的名字不会变。这保证了订单数据的历史一致性。

依赖倒置原则 (DIP)

UserServicePort 定义在 application 层（高层定义接口），UserServiceAdapter 在 infrastructure 层实现它（低层实现接口）。依赖方向始终向内指。

六边形架构 (Hexagonal Architecture)

UserServicePort 是出站端口，UserServiceAdapter 是出站适配器。如果未来从 REST 换成 gRPC，只需换适配器，端口和业务逻辑不变。
